## 001: Rebuild migration.bib from the WordPress export, not via LLM (2026-04-30)

**Decision:** Use `scripts/wp-quotes-matched.json` (the original WordPress quotes export) as the authoritative source when rebuilding broken `migration.bib` entries. Each post carries clean structured fields (`author`, `work_title`, `article_title`, `year`, `locator`, `attribution_text`) plus a `new_cite_key` mapping that lets us deterministically reconstruct the bib entry from data the user originally entered into WordPress.

**Rejected:**
- **Manual triage with hand-fixes**: Too slow for the 614 user-visible migration entries that pass through quotes; would never finish.
- **LLM-assisted enrichment as the primary mechanism**: Burns budget and introduces hallucination risk on ISBNs/page numbers/publishers. Reserve it for the residual ~150 entries that have no WP coverage. The user's first instinct was option 3 (LLM), but locating the WP export changed the calculus — 80% of cases became deterministic.
- **OCLC re-lookup**: That's how we got the original garbage. The OCLC matcher returns wrong records for any cite key whose title fragment overlaps an unrelated work (e.g., Clinton's *What Happened* matched a Polish pneumatic-drive paper).

**Files:** `scripts/rebuild-migration-bib.py`, `scripts/wp-quotes-matched.json`, `scripts/wp-quotes-parsed.json`.
