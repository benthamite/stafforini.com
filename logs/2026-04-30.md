# 2026-04-30: migration.bib cleanup, crossref convention, cite-key normalization

## What was done

End-to-end overhaul of `migration.bib` quality and bib hygiene, triggered by the user not being able to find a BibTeX entry for a Hillary Clinton quote (`/quotes/clinton-artificial-intelligence/`).

### 1. Discovered the broken-migration pattern (early)

- The Clinton entry existed only in `~/My Drive/bibliography/migration.bib`, with all-caps OCLC garbage and `"Place of publication not identified"`. Scanned the 6 bib files to count migration-only cite keys: 746.
- 614 of those are referenced by published quotes; 131 had zero quote references at first glance, but a follow-up audit (using `[cite:@...]` matches in org sources, not just `work=` fields) showed only 1 was truly unreferenced (`Keene2008ChroniclesMyLife`); the others are cited inside quote orgs that attribute their `work` field to a parent.
- Located `scripts/wp-quotes-parsed.json` (2.3 MB) and `scripts/wp-quotes-matched.json` (2.9 MB) — the original WordPress export — covering 1,643 posts with clean `author`, `work_title`, `article_title`, `year`, `locator`, `attribution_text`, plus a `new_cite_key` mapping each post to a migration key. 598 of the 746 migration-only keys had WP coverage; 148 did not (parent series keys, Spanish-language sources).

### 2. Built `scripts/rebuild-migration-bib.py`

Iterated through ~12 rounds of heuristic tightening. Key fixes during iteration:

- **NFC normalization in `get_field`** — bib values stored decomposed (`a` + U+0301); Python `\w+` doesn't match combining marks → mass false-negative plausibility checks.
- **Accent fold in `normalize_word`** (NFKD → ASCII) — `Bernárdez` matches cite key `Bernardez`.
- **No-year cite keys** — `CiceroAmicitia`, `MontaigneEssais`, `HobbesCive` etc.: extract surname as the first CamelCase chunk rather than the whole key.
- **Particle handling removed** — `Miguel de Cervantes` → `Cervantes, Miguel de` (BibTeX convention), not `de Cervantes, Miguel`.
- **Newspaper detection** — added `_NEWSPAPER_WORDS` so the Guardian, Clarín, ZNet, etc. become `@article` / `@online`, not `@incollection`.
- **WP article-title recovery** — WP truncated titles at the first apostrophe (`"One Man's View"` → `"One man"`); regex over `attribution_text` recovers the full quoted span. Also handles missing-closing-quote attributions like Davidson's `'How Is Weakness of the Will Possible?,`.
- **Editor extraction** — `"X (ed.)"` patterns in attribution text → `editor` field. Also handles `quoted in X (ed.), Y, Title` (author=Y, editor=X) for the `PaulEdwards1957AmChristianOther` case (Russell as author, Edwards as editor).
- **Cross-work field-leakage prevention** — added `_original_author_appears_in_wp` guard before preserving any enrichment fields, catching the Ameghino case where the original entry was Caponi's *meta-paper about* Filogenia, not Ameghino's actual Filogenia.
- **Junk publisher / location detection** — strip `"{SIMON} \\& {SCHUSTER}"`, `"EISENBRAUNS"`, `"Place of publication not identified"`, `"S.l."`, etc.
- **Longer-title preference** — when same-work, prefer the original's longer title (`"Bustle"` → `"Life in a bustle: advice to youth"`), with junk-marker guard so AbeBooks/Wikiquote search-result titles aren't preserved.

Final result: 70 entries staged for replacement (down from 96 in the first naive pass), 528 kept as plausible, 148 with no WP coverage.

### 7. Cite-key character normalization

User: cite keys should not have accents or special characters.

Built `scripts/rename-cite-keys.py` — handles bib entries (header + crossref) + bibliographic-notes org file (filename + Custom_ID + ROAM_REFS + `[cite:@key]` + `:work "<slug>"`) + `~/My Drive/notes/**/*.org` cite refs (skipping claude-logs) + `content/quotes/*.md` work field + `static/_redirects` (target retargets + slug-rename redirect appended).

Renamed 31 cite keys:

- 7 with accented letters (Bródy, Comisión, Encyclopédie, Gonzálezricoy, Jørgensen, Müller, Organización)
- 18 with periods (initials like `D.Pennebaker`, org abbreviations like `GOV.UK`, `U.S.GeologicalSurvey`, `Ready.gov`, `St.Jules`, `.ch` TLDs)
- 4 with underscores (`c_uk_2024`, `stavrou_what_2024`, `Locke_USA...`, `no_bear_so_low...`)
- 2 with semicolons (`DaronAcemoglu;...`, `NobleTim;Webster1957...`)

Plus deleted 3 `-dup` orphan entries from `old.bib` (literal duplicates of canonical entries with no incoming references).

## Key findings

- The "131 unused migration-only entries" claim from early in the session was **wrong**. The right metric is `[cite:@key]` references in org files, not `work=` references in quote front matter. Re-audit showed only 1 truly unused entry, with 2 more being crossref parents for cited entries (must keep). Embarrassing initial overclaim; the correction made it clear most migration-only keys are real.
- WP migration's article-title parser broke at apostrophes — multiple posts had truncated titles like `"One man"` (Chomsky's "One Man's View"). The `attribution_text` field is the source of truth here; structured fields are derived.
- OCLC garbage in migration.bib goes deeper than expected: even entries with full publisher / ISBN / DOI metadata were sometimes for *completely unrelated works* matched by a title fragment (Alexander → Indonesian pneumatic drives; Wagner → HIV paper; Wright → wrong Wright). My initial junk-detector caught only obvious "Eisenbrauns + all caps" cases; the real audit needed author-vs-cite-key surname checks plus title-overlap checks.
- Decomposed Unicode in bib values is a real and silent bug — `Cápsulas` stored as `Ca` + U+0301 + `psulas` makes Python's `\w+` split it into `["Ca", "psulas"]`. Not visually distinguishable from composed form.
- 31 cite keys had non-ASCII or special characters that should never have been allowed in the first place. The rename script auto-handles all reference sites (bib + org + notes + redirects) — useful for future renames too.

## Open questions

- The 144 no-WP-coverage migration entries that remain in `migration.bib` are structurally fine but could eventually be promoted to `new.bib` for consistency. Not urgent.
- The crossref convention has only been applied to the cases that surfaced this session (Borges1932, Borges1936, Borges1949Teologos, BioyCasares2002AdPorcos). Other book-keyed quotes (e.g., the remaining `Borges1949Aleph.org` "Germany" quote, plus presumably many quotes in other short-story-collection orgs) still attribute to the book rather than the specific story. A systematic pass would require identifying which story each quote is from — feasible only with LLM assistance per quote.
- `migration.bib` still contains 144 auto-generated entries with `note = "Entry auto-generated from WordPress attribution"` or all-caps publishers that aren't OCLC-junk-shaped but might still be improvable. No urgency.
- The `-dup` deletion suggests there might be other duplicate entries with subtler markers (different cite keys but same PDF / ISBN). Not investigated.
- Several cite keys with unusual `FirstLast` naming convention (`FeliciaAHuppert2005`, `JohnPreston2000`, `PhilipPettit1987`) and one with mismatched accent (`Lukasiewicz1971` for Łukasiewicz) — entries are correct, just non-standard cite keys. Renaming requires updating all references; defer until needed.
